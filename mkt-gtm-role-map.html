产品 x 运营 x Skills 职能链路图

这版按产品发展顺序从上到下展开：每个阶段左侧是产品侧职责，右侧是运营 / MKT 侧职责，中间展示两边如何互相承接。 同时把已安装的 marketing skills 映射到具体工作内容上，方便你在 Codex 里按任务直接调用。 带勾的是你当前正在做的事项；“部分涉及”表示你已经参与但不是完整 owner；“未来要做”表示后续阶段再启动。

Your Current Scope

Early-stage Product GTM Skill Loop

你的工作不是单点运营，而是在产品、市场和增长之间，把研究、定位、内容、上线、转化、数据和复盘变成可重复的 Codex 工作流。

Main Sequence

产品发展顺序

1. 市场与用户研究

用 research、竞品和用户语言决定问题、ICP、关键词和选题。

2. 定位与方案定义

把产品边界、价值主张、定价和反对意见写清楚。

3. 体验与素材交付

把产品流程、prompt、demo、截图、视频和图片变成可发布资产。

4. 内容与上线发布

通过 Blog、社媒、官网、目录、PH、CWS 和外链进入市场。

5. 转化与增长验证

观察注册、onboarding、表单、邮件、销售和互动线索。

6. 数据与复盘迭代

用 tracking、CRO、实验、复盘和留存信号回流产品与内容。

你现在最核心的位置

你不是只在“运营侧”，而是在产品和市场之间做连接。

✓ 关键词 / 用户语言发现 -> 内容主题
✓ 功能边界 -> Messaging 边界
✓ Onboarding 字段 -> Prompt 质量
✓ 产品流程 -> Demo / 截图 / Launch 素材
✓ LinkedIn 评论 -> 判断 ICP / 是否承接
✓ 外链和平台注册 -> 公开入口建设
✓ Daily tracker -> comment suggestion 功能优化
✓ Bug / CR -> 需求池 / 产品迭代
